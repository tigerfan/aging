# P28 家庭诊室——AI辅助居家问诊终端

---

## A. 需求分析文档

### 1. 产品概述与定位

**家庭诊室**是一款集成多项生理检测功能与远程问诊能力的居家医疗终端，外形为大屏智能台机（类似自助挂号机的家庭版），集成血压计、体温计、血氧仪、心电图导联、皮肤病变摄像头等5种医疗级传感器，老人一次性完成基础生命体征采集后，可连接医院医生进行图文或视频远程问诊，并由AI助手自动整理病史、检测数据，为医生提供结构化的就诊前摘要，大幅提升远程问诊效率和质量。

**蓝海点**：互联网医院在中国快速发展，但"居家自采数据"环节是最大短板——医生不相信患者自测数据的准确性；本产品通过医疗级传感器+标准化采集流程，填补居家检测数据可信度的关键缺口。

### 2. 目标用户画像

| 维度 | 描述 |
|------|------|
| 核心用户 | 65–85岁，行动不便或交通不便，有慢病随访需求（高血压/糖尿病/慢阻肺）的老年人 |
| 次要用户 | 偏远农村/山区，就医极不方便的老年患者 |
| 使用场景 | 慢病复诊（每月1–2次），病情有变化时临时就诊，替代部分面对面门诊 |
| 医疗资源限制 | 三甲医院挂号难、候诊时间长、老年人就医体验差 |

### 3. 市场痛点与机会

- 中国老年人慢病随访率极低，主要原因是就医麻烦（交通/候诊/认知障碍）
- 互联网医院（阿里健康/京东健康）：主要依赖患者文字/照片，诊断质量受限
- 居家可用的医疗级多参数检测设备几乎空白（单品散乱，无整合）
- 医生普遍抱怨远程问诊缺乏体征数据支持，家庭诊室可解决这一核心痛点
- "医疗级数据可信度"是远程问诊市场最大的差异化竞争点

### 4. 功能性需求（FR）

**FR-01 医疗级多参数检测（集成）**
- 血压：充气袖带，±3mmHg精度（符合ISO 81060-2）
- 体温：非接触式额温（精度±0.2°C）
- 血氧SpO2：指夹式，±2%精度（符合ISO 80601-2-61）
- 单导联心电图：双手握式电极，采集30秒ECG（I导联，AF筛查）
- 皮肤摄像头：10倍放大微距镜头（皮疹、伤口、眼睑、咽喉等特写）
- 体重：脚踏式体重秤（外置，蓝牙连接）

**FR-02 AI病史整理助手**
- 检测完成后，AI语音问卷收集主诉（"今天主要哪里不舒服？多久了？"）
- 与历史检测数据对比，自动生成变化摘要（"血压比上次升高15mmHg，心率偏快"）
- 按SOAP格式（Subjective/Objective/Assessment/Plan）生成就诊前摘要推送给医生
- 慢病状态：自动标注"对比上月：血糖控制改善/HbA1c预测"

**FR-03 远程问诊连接**
- 对接互联网医院（平台API对接或内建互联网医院资质）
- 检测数据一键推送给接诊医生（标准化HL7 FHIR格式）
- 高清视频问诊（1080p摄像头，专业补光），医生可实时看到老人面色
- 皮肤特写：将微距摄像头画面实时分享给医生

**FR-04 检测操作全引导**
- 每项检测步骤有语音+动画图示引导（"请将手臂放平，手心向上..."）
- 操作错误即时纠正（"血压袖带位置偏高，请往下调整"）
- 全程完成时间约8–10分钟（完成5项检测+AI问诊）

**FR-05 健康档案与家属共享**
- 所有检测结果自动存入个人健康档案
- 家属APP实时查看检测结果
- 对接国家健康信息平台（电子健康档案），历次就诊记录整合
- 急救档案：紧急情况下，医生可一键查看患者用药史、过敏史、慢病史

### 5. 非功能性需求（NFR）

| 维度 | 指标 |
|------|------|
| 血压精度 | ±3mmHg（ISO 81060-2 A级） |
| 视频问诊质量 | 1080p/30fps，延迟<200ms，支持弱网降级（3Mbps以上可用） |
| 界面字体 | ≥32pt，颜色对比度≥7:1 |
| 操作完成率 | 老年人独立完成5项检测+问诊比例>80% |
| 数据传输 | TLS 1.3加密，符合卫生信息传输标准 |
| 屏幕 | 15.6英寸触控屏（站立或坐姿均可操作） |

### 6. 约束条件与合规要求

- 集成血压/心电图须通过NMPA **II类医疗器械**认证
- 互联网医院执照：需获得互联网诊疗资质（或与持照互联网医院合作）
- 医生端：只有执业医师账号可接入处方开具功能
- 数据合规：医疗数据按《个人信息保护法》和《网络安全法》处理，服务器在中国境内
- 处方开具：仅限慢病复诊（不适用首诊处方），符合国家互联网诊疗政策

## B. 详细设计方案

### 1. 系统架构

```
[家庭诊室终端]
     ├── 15.6英寸触控屏
     ├── 1080p高清摄像头（视频问诊+AI面诊）
     ├── 血压袖带（内置，自动充放气）
     ├── 额温枪（非接触，单次）
     ├── 指夹SpO2（可拔插）
     ├── ECG电极握片（双手握式）
     ├── 微距皮肤摄像头（可旋转）
     ├── 主控SBC（i5/Ryzen等级，运行AI模型）
     └── 4G LTE + WiFi 6
          │ HTTPS / WebRTC
          ▼
[互联网医院平台]
     ├── 医生接诊工作站
     ├── 视频问诊系统
     └── 电子病历系统
          ▼
[家属APP + 国家健康平台]
```

### 2. 硬件设计方案

**2.1 主机**
- 计算核心：Intel N100（低功耗x86，8GB RAM，256GB SSD）
- 操作系统：Windows 11 LTSC（稳定性高，医疗设备驱动兼容性好）
- 显示：15.6英寸IPS触控（1920×1080），防眩光，亮度≥400nit
- 摄像头：索尼IMX258（1080p主摄）+ IMX477（12MP，微距皮肤摄像头，带LED补光）
- 扬声器：2×8W，TTS语音清晰，确保操作引导可听清

**2.2 医疗传感模块**
- 血压：欧姆龙OEM血压袖带模组（NMPA二类认证量级，集成气泵/传感器/算法）
- SpO2：Masimo Rainbow SET指夹（医疗级，符合ISO 80601-2-61）
- ECG：TI ADS1292R ECG前端模拟，双不锈钢握片电极
- 额温：Melexis MLX90632（非接触红外，精度±0.2°C）
- 微距摄像头：RaspCam HQ Module + 倍率10× macro镜头，LED冷光圈

**2.3 结构设计**
- 台式一体机形态（类似医院自助机小型化），底座稳定
- 袖带收纳：侧面隐藏式挂钩，使用时拉出
- 高度可调：支架升降50–90cm，适应坐姿和轮椅用户

### 3. 软件设计方案

**3.1 检测引导系统**
```
检测流程（自动化脚本）：
Step 1: 血压（语音："请将左手穿入袖带..." → 自动充气测量）
Step 2: 体温（语音："请额头靠近温度传感器" → 自动读数）
Step 3: 血氧（语音："请将右手食指放入指夹" → 30秒稳定读数）
Step 4: 心电（语音："请双手分别握住两侧金属握片，保持5秒不动" → 30s ECG）
Step 5: 皮肤摄像（语音引导对焦 → 患者说"拍"→ 一键拍摄）
Step 6: AI问诊（"检测已完成，请告诉我今天哪里不舒服？"）
```

**3.2 AI病史摘要生成**
```python
def generate_pre_consult_summary(vitals, symptoms, history):
    """生成SOAP格式就诊前摘要"""
    
    # Subjective（主诉）
    subjective = llm_extract_chief_complaint(symptoms)
    
    # Objective（客观数据）
    objective = f"""
    血压: {vitals.sbp}/{vitals.dbp} mmHg {'↑' if vitals.sbp>140 else ''}
    心率: {vitals.hr} 次/分
    体温: {vitals.temp}°C
    SpO2: {vitals.spo2}%
    心电图: {vitals.ecg_interpretation}
    对比上次（{days_since_last}天前）：{generate_changes(vitals, history)}
    """
    
    # Assessment（AI初步评估，仅供医生参考）
    assessment = ai_triage_model.assess(vitals, symptoms, history)
    
    summary = format_soap(subjective, objective, assessment)
    return summary
```

**3.3 视频问诊技术栈**
- WebRTC（P2P，低延迟）
- 网络自适应（弱网自动降码率至720p/480p）
- 端到端加密（DTLS-SRTP）
- 皮肤摄像头画中画：医生端可请求切换到微距摄像画面

### 4. 核心算法设计

**4.1 ECG快速分析（30秒I导联）**
- Pan-Tompkins R波检测
- 房颤初筛（RR间期不规则性，Lorenz图）
- PR间期/QRS宽度/ST段测量
- 报告："正常窦律"/"疑似房颤（需确认）"/"QRS增宽（需确认）"

**4.2 AI面诊辅助（主摄摄像头）**
- 肤色分析：结膜是否苍白（贫血提示）、面部潮红（高热提示）
- 口唇颜色：发绀检测（SpO2低）
- 以上为辅助标志，推送给医生参考，非诊断

**4.3 皮肤病变初步筛查**
- 微距照片输入：SkinVision算法（良性/可疑/建议活检分级）
- 结果显示给医生，不直接告知患者诊断
- 数据用于医生辅助判断，最终决策由医生做出

### 5. 用户交互设计

**完整就诊流程（约15分钟）**
```
老人坐到终端前 → 人脸识别自动登录
→ "您好，今天需要看什么科？"→ 选择科室（高血压复诊/感冒/皮肤）
→ 自动进入相应检测流程（5项检测，8分钟）
→ AI语音问诊（2分钟）
→ 系统：正在为您寻找医生...（1–3分钟排队）
→ 视频接通：医生已看到所有检测数据，直接开始问诊
→ 问诊结束：电子处方/检查建议自动推送到手机+药店
```

### 6. 商业化路径与技术风险

**商业化路径**
- **硬件**：4,800元（面向家庭购买）；3,200元/台（B端养老院集采）
- **问诊服务**：每次问诊30–80元（根据医生级别）；包月会员199元/月（无限次慢病复诊）
- **渠道**：与互联网医院平台合作（京东/阿里/平安好医生），硬件售卖+平台导流
- **政府合作**：基层医疗信息化项目，农村卫生室升级配置

**技术风险评估**

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 血压袖带操作（老人独立穿戴困难） | 高 | 动画+语音全程引导，自动检测袖带松紧（提示"太松请重新佩戴"） |
| 医生对居家数据可信度疑虑 | 高 | 显示设备认证信息和检测质量评分（"本次检测质量：优良"）增强信任 |
| 互联网诊疗政策限制 | 中 | 严格限于慢病复诊，首诊不开处方，与具有资质的合作平台合规运营 |
| 硬件集成可靠性（多传感器故障） | 中 | 模块化设计（各传感器独立可更换），远程诊断（APP显示各模块状态） |
