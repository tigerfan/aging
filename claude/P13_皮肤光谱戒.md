# P13 皮肤光谱戒——无创血糖连续监测戒指

---

## A. 需求分析文档

### 1. 产品概述与定位

**皮肤光谱戒**是一款通过近红外光谱技术（NIR）实现无创连续血糖监测的智能戒指。多波长NIR光穿透手指指腹皮肤，与血液葡萄糖分子发生特征吸收，结合个性化机器学习模型从光谱信号中提取血糖浓度，为老年糖尿病患者提供无采血的日常监测体验。配合APP实现血糖趋势分析、低血糖预警、饮食关联分析和医生随访数据共享。

**蓝海点**：全球1.4亿中国糖尿病患者，采血痛苦是依从性最大障碍；CGM（连续葡萄糖监测）需皮下植入，老年人接受度低；纯光学无创方案是行业圣杯级需求。

### 2. 目标用户画像

| 维度 | 描述 |
|------|------|
| 核心用户 | 60–80岁，2型糖尿病患者，需日常血糖监测但厌倦采血 |
| 次要用户 | 糖尿病前期人群，希望了解饮食对血糖影响 |
| 使用场景 | 全天佩戴，餐前/餐后自动检测，睡眠期监测血糖谷值 |
| 医疗背景 | 内分泌科医生随访，配合HbA1c定期复查 |
| 痛点程度 | 极高，每日3–5次采血是患者最主要的依从性障碍 |

### 3. 市场痛点与机会

- 中国糖尿病患者1.4亿，血糖监测依从性极低，并发症发生率居高不下
- 传统指尖采血：疼痛、繁琐、耗材贵（试纸5–8元/片）
- 雅培/德康CGM需皮下植入传感器，老年人接受度低，费用每月600–1,200元
- NIR光谱+深度学习精度持续提升，个性化校准模型是商业化关键突破口
- Apple Watch无创血糖被广泛期待，市场教育成熟，先行者有巨大先发优势

### 4. 功能性需求（FR）

**FR-01 多波长NIR光谱采集**
- 集成10个NIR LED（波长750–1350nm，覆盖葡萄糖特征吸收峰1050nm/1150nm）
- InGaAs光电探测器（对NIR灵敏度高）
- 每15分钟自动检测一次，餐后加密至每5分钟
- 压力传感器确保接触力一致（影响光谱重复性的关键因素）

**FR-02 血糖估算与个性化模型**
- 精度目标：MARD≤15%（与医院静脉血糖对比）
- 个性化校准：首次2周内收集50对以上NIR光谱+指尖血糖，训练专属XGBoost模型
- 模型每30天更新（补偿皮肤老化、体重变化带来的个体漂移）
- 低血糖预警：估算值<3.9mmol/L时振动+推送告警

**FR-03 血糖管理APP**
- 24小时血糖趋势曲线，未来1小时预测
- 饮食记录（AI图像识别估算食物GI指数），分析饮食对血糖影响
- 运动后血糖下降曲线分析
- 月度TIR（目标范围内时间）报告，AGP格式导出给医生

**FR-04 基础健康监测**
- 心率、SpO2、皮肤温度（作为血糖模型协变量）
- 步数/活动量（6轴IMU）
- 睡眠期血糖谷值自动记录

**FR-05 医生端对接**
- CGM标准AGP报告格式导出
- 低血糖事件统计、血糖变异系数（CV）
- 医生可设置个性化告警阈值

### 5. 非功能性需求（NFR）

| 维度 | 指标 |
|------|------|
| 血糖精度 | MARD≤15%（目标），初期≤20%可接受 |
| 续航 | ≥48小时，磁吸充电，30分钟充至70% |
| 防水 | IP68（可洗手、游泳） |
| 重量 | <15g |
| 皮肤接触 | Ti-6Al-4V钛合金内表面，无镍过敏 |
| 数据安全 | 医疗数据端对端加密，用户可申请完整删除 |

### 6. 约束条件与合规要求

- 作为**II类医疗器械**申报NMPA，精度须满足ISO 15197标准
- 产品必须声明：这是"参考监测"工具，不能替代临床血糖检验用于剂量决策
- 低血糖告警阈值须保守，避免漏报
- 个人光谱数据严禁用于第三方商业分析

## B. 详细设计方案

### 1. 系统架构

```
[皮肤光谱戒]
     ├── 10波长NIR LED阵列（750–1350nm）
     ├── InGaAs光电探测器
     ├── 接触压力传感器
     ├── PPG + SpO2 + 皮肤温度传感器
     ├── 6轴IMU
     └── MCU + BLE 5.3
          │ BLE
          ▼
[手机APP（边缘AI推理）]
     ├── 个人血糖模型（XGBoost，本地运行）
     ├── 趋势图表与预测
     └── 医生数据共享
          ▼
[云端]
     ├── 个性化模型训练服务
     ├── AGP报告生成
     └── 医生随访平台
```

### 2. 硬件设计方案

**2.1 光学系统**
- NIR LED：10个VCSEL，波长750/850/940/1050/1100/1150/1200/1300/1350nm，分时复用
- 光电探测器：Si（700–1100nm）+ InGaAs（1000–1600nm）双探测器
- 光路：内反射式漫反射模式，LED环绕、探测器居中
- 接触均匀化：内置微型气囊（0.5–1.5kPa），确保指面与光窗均匀贴合

**2.2 主控与电源**
- MCU：Nordic nRF5340（低功耗，BLE 5.3）
- ADC：ADS1299（24bit，多通道光电信号采集）
- 电池：50mAh锂聚合物电池（嵌入戒圈，厚度3mm），磁吸无线充电8小时充满
- 存储：4MB SPI Flash（本地缓冲48小时数据）

**2.3 结构设计**
- 外壳：CNC钛合金，蓝宝石光学窗口（高透NIR，耐划）
- 尺寸：提供S/M/L三款（内径17/19/21mm），戒面高度5mm
- 防水：超声波焊接全密封，IP68

### 3. 核心算法设计

**3.1 光谱预处理**
```
原始吸收谱 A = -log(I/I₀)
    ↓ Savitzky-Golay二阶导数（消除基线漂移）
    ↓ 标准正态变量变换SNV（消除散射效应）
    ↓ 温度补偿（皮肤温度作为线性协变量扣除）
    ↓ 10维特征向量 → 模型输入
```

**3.2 个性化校准模型**
```python
def calibrate_personal_model(spectrum_list, glucose_list):
    X = [preprocess(s) for s in spectrum_list]
    y = glucose_list
    model = XGBRegressor(n_estimators=100, max_depth=4)
    model.fit(X, y)
    mard = mean_abs_relative_diff(model.predict(X), y)
    return model if mard < 0.20 else None  # 精度不足时提示继续采集

def predict_glucose(spectrum, temperature, hr, model):
    features = preprocess(spectrum, temperature, hr)
    glucose = model.predict([features])[0]
    return glucose
```

**3.3 低血糖预测**
- 过去30分钟下降速率 > 0.1mmol/L/min 且当前值<5.0mmol/L → 预警
- 当前值<3.9mmol/L → 紧急振动+声音+家属推送

### 4. 用户交互设计

**佩戴与校准流程**
```
佩戴戒指（对准指腹光窗）→ APP自动连接 → 信号质量检测
→ 显示"今日血糖监测已开启"
→ 餐前/餐后自动检测并推送趋势图
→ 低血糖预警：戒指振动 + "您的血糖偏低，请及时补充糖分"
```

**APP界面设计**
- 主页：大字体当前血糖值 + 24h趋势曲线
- 颜色编码：绿（正常）、黄（偏高/偏低）、红（告警）
- 饮食日记：拍照即识别，自动记录GI值
- 月报：TIR饼图 + 与上月对比改善幅度

### 5. 商业化路径与技术风险

**商业化路径**
- **定价**：戒指1,980元 + 校准服务套件（含初始指尖血糖试纸50片）
- **订阅**：129元/月（云端模型维护 + 医生随访 + 月度报告）
- **渠道**：内分泌科医院合作推荐、糖尿病患者社群、医疗器械电商

**技术风险评估**

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| NIR精度受皮肤水分/运动影响 | 高 | 静息状态检测，运动期暂停，压力均匀化设计 |
| 个性化模型需50对以上配对数据 | 高 | 前2周引导校准，提供便携指尖血糖仪配套赠送 |
| NMPA审批周期长（3–5年） | 中 | 先以"健康监测"类产品上市，积累临床数据支撑后续注册 |
| 光学器件小型化成本高 | 中 | 与晶圆厂定制集成VCSEL阵列芯片，量产后成本可控 |
