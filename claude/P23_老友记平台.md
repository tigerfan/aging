# P23 老友记平台——老年人兴趣社群智能匹配平台

---

## A. 需求分析文档

### 1. 产品概述与定位

**老友记平台**是一款专为老年人设计的兴趣社群匹配与线上线下活动组织平台，通过AI兴趣画像分析，将同城同兴趣的老年人精准匹配，自动组织棋牌局、健身小组、手工课、读书会等小型圈子活动，并配合专属智能助手（老友机器人）降低老年人使用数字社交平台的门槛。产品软硬件结合：软件平台负责匹配和组织，专属桌面终端提供极简化社交入口，解决老年人智能手机操作困难的痛点。

**蓝海点**：现有社交APP（微信、抖音）被老年人大量使用，但针对"陌生老年人之间建立新连接"的社群产品几乎空白；现有老年大学、棋牌室等线下载体缺乏数字化匹配手段。

### 2. 目标用户画像

| 维度 | 描述 |
|------|------|
| 核心用户 | 60–75岁，退休后社交圈收缩，希望结交新朋友的城市老年人 |
| 次要用户 | 刚迁居新城市（跟随子女）、丧偶、退休不久的老年人 |
| 兴趣分布 | 棋牌（麻将/象棋）、广场舞、健步走、书法/绘画、园艺、戏曲、手工 |
| 痛点 | 退休后社交圈大幅萎缩，找不到同龄朋友，线下活动信息获取困难 |
| 技术水平 | 会用微信发消息，但对陌生APP学习门槛高 |

### 3. 市场痛点与机会

- 中国退休老年人超2亿，社交孤立是仅次于身体衰退的最大生活质量杀手
- 现有老年社交（老年大学/广场舞）：地理局限性强，缺乏系统匹配
- 老年版抖音/快手：内容消费型，不能促成真实社交连接
- 社区养老中心：服务能力有限，无法为个性化兴趣社群提供充足内容
- 企业退休人员协会、居委会：组织方式传统，缺乏数字化工具

### 4. 功能性需求（FR）

**FR-01 兴趣画像建立**
- 注册时引导问卷：兴趣爱好（图片+文字选择，无需打字）、作息时间、活动距离偏好（步行/公交/开车）
- AI持续学习：根据参加活动记录、互动内容、评价，动态更新兴趣权重
- 画像维度：兴趣类别、活跃时间段、出行能力、社交偏好（大群/小圈/一对一）

**FR-02 智能社群匹配**
- 每周推荐3–5个"新圈子"（基于兴趣相似度+地理距离+时间匹配）
- 圈子规模：每个小圈6–12人（有研究表明这是老年人社交最舒适的规模）
- 匹配算法：协同过滤 + 地理围栏（步行距离优先）+ 时间槽重叠
- 系统自动组织：匹配后AI助手（老友机器人）代劳组织协调（场地/时间）

**FR-03 活动组织**
- 线下活动：系统自动在附近合作场地（棋牌室/咖啡厅/公园）预订场地
- 活动提醒：短信+终端提醒（"明天下午2点，您的棋友圈在朝阳公园第3亭"）
- 活动记录：照片打卡，自动发给家属（"爸爸今天参加了象棋圈活动"）
- 安全评估：首次见面活动安排在公共场所，系统提供安全提示

**FR-04 老友机器人（社交AI助手）**
- 在圈子内帮助老人发起话题、回应消息（老人语音说，机器人代发）
- 提醒老人圈子内朋友的生日、共同纪念日
- 活动协调：代老人回复是否参加、帮助调整时间（"我帮您告诉大家您下午3点才有空"）

**FR-05 桌面终端（专属硬件）**
- 8英寸彩屏，显示今日活动/新消息/新朋友推荐
- 物理按键：3个常用操作（参加活动/联系朋友/查看照片）
- 语音交互：说话即可操作，无需打字
- 微信/电话联动：与老人现有微信好友打通，不需要迁移社交关系

### 5. 非功能性需求（NFR）

| 维度 | 指标 |
|------|------|
| 匹配准确度 | 用户对推荐圈子的接受率>40% |
| 终端操作步骤 | 任何核心功能≤3步完成 |
| 内容安全 | 防诈骗过滤（识别以"交友"为幌子的销售诱导内容），实名制 |
| 活动成功率 | 组织活动到场率>70%（小圈子制度保障） |
| 响应时间 | AI助手对话响应<2秒 |

### 6. 约束条件与合规要求

- **实名制**：所有用户须身份证实名认证（保护老年人免受诈骗）
- **防诈骗机制**：自动检测微商、投资推销、保健品传销内容，发现立即封号
- **隐私**：精确位置仅用于活动组织，不对其他用户显示家庭住址
- **未成年内容隔离**：平台定向65岁以上用户，防止未成年账户混入

### 3. 核心算法设计

**3.1 社群匹配算法**
```python
def match_social_circle(user, candidate_pool, max_size=10):
    scores = []
    for candidate in candidate_pool:
        # 兴趣相似度（Jaccard系数）
        interest_sim = jaccard_similarity(user.interests, candidate.interests)
        
        # 地理距离（步行时间）
        distance = walking_distance(user.location, candidate.location)
        geo_score = max(0, 1 - distance/30)  # 30分钟步行为满分
        
        # 时间槽重叠（可用时间重叠小时数）
        time_overlap = compute_time_overlap(user.available_slots, 
                                            candidate.available_slots)
        
        # 综合评分
        score = interest_sim*0.5 + geo_score*0.3 + time_overlap*0.2
        scores.append((candidate, score))
    
    top_matches = sorted(scores, key=lambda x: x[1], reverse=True)[:max_size]
    return [m[0] for m in top_matches]
```

**3.2 活动自动组织机器人**
- 匹配成圈 → 机器人发出邀请（"我是老友助手，帮您和几位棋友组织下棋活动"）
- 收集所有人的可用时间（LLM解析自然语言时间表达："周三下午都行"）
- 计算最优时间槽（最大公约时间）
- 自动在合作场地预订时间（调用第三方场地API）
- 发送确认通知给所有参与者

**3.3 防诈骗内容检测**
```python
def detect_fraud_content(message_text):
    fraud_keywords = ['投资', '理财', '高收益', '加盟', '代理',
                      '保健品', '专利技术', '内部名额', '限时优惠']
    keyword_count = sum(1 for kw in fraud_keywords if kw in message_text)
    
    if keyword_count >= 2:
        # 触发LLM二次语义判断
        is_fraud = llm_fraud_classifier(message_text)
        if is_fraud:
            return True, "疑似营销诈骗内容"
    return False, None
```

### 4. 用户交互设计

**老年人端（终端）**
```
按"今日活动"键 → 显示"明天下午，您的棋友圈在朝阳公园"
                  → 确认参加/请假（语音说"我去"/"我有事"）
                  → 自动通知其他人

按"联系朋友"键 → 显示最近联系的5位朋友头像（大图）
                 → 说"打给老王" → 一键通话

按"新消息"键 → 朋友留言/活动通知/新朋友推荐 → 逐条语音播报
```

**匹配推荐流程**
```
系统推荐："发现3位兴趣相近的棋友，是否想认识他们？"
老人说"好的" → 系统介绍三位棋友（头像+兴趣描述，不显示地址）
→ "帮您安排一次茶馆象棋局，有兴趣吗？"
→ 自动预订，发送活动通知
```

### 5. 商业化路径与技术风险

**商业化路径**
- **终端硬件**：680元（可替换为APP模式，终端为可选项）
- **平台订阅**：免费基础版（每月5次活动匹配）/ 69元/年（无限匹配+场地优惠）
- **场地分成**：与棋牌室/茶馆/公园驿站合作，每次预订平台抽成15%
- **B端**：社区养老服务中心数字化工具，政府购买服务潜力大
- **广告**：老年人兴趣相关品牌（书法、棋类、园艺工具）精准投放（非骚扰性）

**技术风险评估**

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 老年人对陌生人社交顾虑（防骗意识） | 高 | 实名制+社区认证+首次见面公共场所强制要求，建立安全感 |
| 冷启动（早期用户密度不够，匹配困难） | 高 | 先聚焦1–2个城市深耕，与居委会/社区中心合作导入种子用户 |
| 诈骗分子混入平台 | 高 | 实名制+AI内容检测+用户举报+人工审核三道防线 |
| 活动组织失败（有人爽约） | 中 | 小圈子制度（6–10人），设置信用评分，多次爽约降低匹配优先级 |
