# P03 时光相框——情绪感知智能记忆相册

---

## A. 需求分析文档

### 1. 产品概述与定位

**时光相框**是一款专为阿尔茨海默症患者及其家庭设计的AI智能相册设备。它能自动识别家庭老照片中的人物，结合情感计算技术感知患者当前情绪状态，主动播放与当前情绪相匹配的家庭记忆故事（语音合成讲述）、家人视频留言和怀旧音乐，提供情感陪伴与认知刺激，同时向远端家属实时推送患者情绪状态。

**产品定位**：阿尔茨海默症早中期患者居家陪护辅助设备，降低照护者负担，维系家庭情感联结。

### 2. 目标用户画像

| 维度 | 描述 |
|------|------|
| 主要使用者 | 60–85岁，轻中度阿尔茨海默症或其他痴呆症患者 |
| 购买决策者 | 成年子女，出于减轻照护压力、维系父母情感的动机 |
| 照护者 | 专业护理员或家庭成员，需要设备操作简便 |
| 使用环境 | 老人卧室、客厅床头柜，长期通电放置 |
| 家属特点 | 异地居住（候鸟家庭），无法每日陪伴 |

### 3. 市场痛点与机会

- 中国现有阿尔茨海默症患者超1,000万，照护缺口巨大，专业护理人员极度短缺
- 回忆疗法（Reminiscence Therapy）是国际公认的非药物痴呆干预手段，但实施需要大量人力
- 现有智能音箱（天猫精灵、小度）无法感知情绪、无法针对认知障碍者专门设计
- 家属对远程了解老人情绪状态有强烈需求，现有摄像头方案侵犯隐私
- 家庭照片数字化需求旺盛，相册形态天然获得老年人信任感

### 4. 功能性需求（FR）

**FR-01 照片智能管理**
- 家属通过APP上传家庭老照片（支持纸质照片扫描上传）
- AI自动识别照片中人物（家庭成员标注），标注时间、地点、事件信息
- 照片库按时间轴、人物、事件自动分类整理

**FR-02 情绪感知**
- 摄像头（隐藏式，不明显）实时采集面部表情
- AI分析情绪状态（平静/高兴/悲伤/焦虑/困惑），更新频率每5分钟
- 结合时间段（早/午/晚）和历史规律，预测当前情绪走势

**FR-03 主动内容推送**
- 根据情绪状态自动选择内容类型：
  - 高兴/平静 → 播放家庭记忆故事、家人视频留言
  - 悲伤/焦虑 → 播放怀旧音乐、家人录制安慰语音
  - 困惑/激动 → 播放家人面孔识别引导（"这是你的女儿小红"）
- 内容切换间隔≥30分钟，避免过度刺激

**FR-04 记忆故事讲述**
- 家属录入家庭故事文本，系统用语音合成（TTS）以老人熟悉的家乡口音播讲
- 支持家属亲自录制语音故事（保留真实音色更有情感共鸣）
- 故事时长30秒–3分钟，配合相框显示对应照片

**FR-05 远端家属联结**
- 家属APP收到每日情绪摘要推送（文字+表情图标，非视频，保护隐私）
- 家属可在APP录制"留言"，设定时间段在相框上播放
- 双向语音通话（家属发起，相框自动接听）

**FR-06 异常行为告警**
- 持续超过2小时处于焦虑/激动状态时，推送家属告警
- 长时间无活动检测（超过设定时间未在相框前出现），推送安全确认提醒
- 与家庭智能门锁联动，防止夜间走失（仅提醒，不锁门）

### 5. 非功能性需求（NFR）

| 维度 | 指标 |
|------|------|
| 易用性 | 老人无需任何操作，设备全自动运行 |
| 响应时间 | 情绪检测到内容播放<3秒 |
| 画面质量 | 10.1英寸IPS屏，分辨率1920×1200，亮度可调（100–400nit） |
| 音频 | 双5W全频扬声器，频率响应100Hz–15kHz，清晰语音还原 |
| 可靠性 | 7×24小时连续运行，MTBF>15,000小时 |
| 隐私保护 | 面部视频不上传云端，仅情绪标签上传；本地摄像头镜头盖物理开关 |
| 设计感 | 实木相框外观，与家庭装修融合，不像医疗设备 |

### 6. 约束条件与合规要求

- 定位为**消费电子产品**（非医疗器械），不作疾病诊断声明
- 面部识别功能须获得用户（家属代理）明确授权，符合《个人信息保护法》
- 照片/故事内容存储采用端对端加密，平台人员无权访问
- 摄像头指示灯：工作时必须亮起绿色LED，不得隐藏工作状态

## B. 详细设计方案

### 1. 系统架构

```
[时光相框设备]
     ├── 10.1寸IPS触控屏（家属配置时使用）
     ├── 隐藏式摄像头（情绪感知，镜头盖物理开关）
     ├── 双5W扬声器
     ├── 麦克风阵列（语音通话）
     └── 本地算力：NPU加速（情绪识别边缘推理）
          │ WiFi 6 / 以太网
          ▼
[家庭云服务]
     ├── 照片库（加密存储，AI标注服务）
     ├── 内容调度引擎（情绪→内容匹配）
     ├── TTS语音合成（家乡方言支持）
     └── 家属通知服务
          │ HTTPS
          ▼
[家属APP（iOS/Android）]
     ├── 照片上传与管理
     ├── 故事录制与编辑
     ├── 情绪摘要接收
     └── 视频/语音通话发起
```

### 2. 硬件设计方案

**2.1 显示与音频**
- 显示：10.1英寸IPS，分辨率1920×1200，亮度100–500nit，防眩光镀膜
- 触控：电容式触控（家属初始配置用），老人日常不需触控操作
- 音频：双5W全频扬声器，DSP音效处理，专为语音播放调音

**2.2 计算模块**
- 主控SoC：瑞芯微RK3588S（八核，集成6TOPS NPU）
- 内存：4GB LPDDR4X + 64GB eMMC（本地照片库+模型）
- 扩展：MicroSD卡槽（最大支持256GB，存储家庭大容量照片库）

**2.3 摄像头与传感**
- 摄像头：200万像素，固定焦距广角（100°），内置850nm红外补光（夜间情绪感知）
- 物理镜头盖：推拉式，遮盖时摄像头完全不工作（硬件断路）
- 光线传感器：ALS调节屏幕亮度，夜间自动调暗

**2.4 结构设计**
- 外观：实木（榉木/胡桃木）相框+亚克力背板，复古工艺
- 尺寸：28×20×3cm，竖/横屏均可放置
- 供电：12V/2A DC适配器，内置2,000mAh锂电（短暂停电保护，续航30分钟）
- 接口：USB-A×1（U盘导入照片）、3.5mm耳机、百兆以太网

### 3. 软件设计方案

**3.1 设备端软件（基于Android 12定制）**
```
├── 情绪感知服务（后台常驻）
│     ├── 人脸检测（MobileNet-SSD，<10ms/帧，每5分钟深度分析）
│     ├── 情绪识别模型（AffectNet预训练，老年人数据微调，7类情绪）
│     └── 情绪平滑：5次检测取中位数，避免单帧误判
├── 内容播放引擎
│     ├── 内容调度器（情绪状态→内容规则引擎）
│     ├── 照片轮播（Ken Burns动效，平缓切换）
│     ├── 音频播放（TTS/录音/音乐混合）
│     └── 通话模块（SIP协议，WebRTC）
├── 本地照片AI服务
│     ├── 人脸识别（家庭成员标注）
│     ├── OCR（老照片背面文字识别）
│     └── 场景分类（室内/户外/节日等）
└── 设备管理服务（OTA更新、日志上传、健康检测）
```

**3.2 云端服务**
```
├── 照片存储服务：分片加密上传（AES-256）、CDN加速
├── AI标注流水线：人脸聚类→人工确认→自动标注
├── 内容调度引擎：
│     规则库：{情绪状态, 时间段, 近期已播内容} → {内容类型, 具体内容ID}
│     机器学习层：基于历史用户反应（面部表情改善情况）强化学习优化
├── TTS服务：支持普通话+20种方言，老年人语速适配（0.8×正常语速）
└── 通知服务：情绪摘要生成（NLP）、APP推送、SMS告警
```

**3.3 家属APP**
```
├── 相册管理：批量上传、AI标注审核、故事编辑器
├── 留言录制：语音/视频留言，设置播放时间段（如：早8点播放早安留言）
├── 情绪看板：日历热力图（每日情绪状态）、本周趋势折线图
├── 实时通话：一键发起语音/视频通话
└── 设置中心：内容偏好、告警阈值、隐私开关
```

### 4. 核心算法设计

**4.1 老年人情绪识别模型**
- 基础模型：在AffectNet（100万张标注图像）上预训练的EfficientNet-B0
- 域适配：采集300名老年人（60–85岁）的情绪表情数据进行微调
- 输出：7类情绪（高兴/平静/悲伤/焦虑/困惑/激动/疲倦）+ 置信度
- 特殊处理：老年人面部特征差异（皱纹增加、表情幅度减小），增加面部关键点权重

**4.2 内容-情绪匹配规则引擎**

| 情绪状态 | 优先内容 | 次选内容 | 禁止内容 |
|----------|----------|----------|----------|
| 高兴 | 欢快家庭记忆故事 | 轻音乐 | 无 |
| 平静 | 自然风景照片 | 轻柔音乐 | 激动内容 |
| 悲伤 | 家人安慰录音 | 怀旧音乐 | 悲伤故事 |
| 焦虑 | 家人熟悉声音引导 | 舒缓音乐 | 复杂信息 |
| 困惑 | 家人面孔识别引导 | 简单重复内容 | 新内容 |
| 激动 | 平静语音引导（护理员语气） | 降噪环境音 | 视频通话 |

**4.3 内容个性化强化学习**
- 状态：当前情绪类别
- 动作：选择内容类别
- 奖励：播放后5分钟内情绪是否改善（改善=+1，恶化=-1，不变=0）
- 算法：表格Q-Learning（状态空间小，可解释性强，适合医疗场景）

**4.4 人脸聚类与家庭成员识别**
- 离线聚类：FaceNet嵌入向量提取，DBSCAN聚类，自动发现家庭成员群组
- 在线识别：余弦相似度比对已标注人脸库（阈值0.6），置信度<80%时不显示姓名
- 隐私：人脸向量在本地设备计算，不上传原始人脸图像

### 5. 用户交互设计

**5.1 老人端（全自动，无需操作）**
- 上电即运行，无界面操作
- 设备状态：绿色圆点常亮（正常工作）、橙色（通话中）、红色（异常）
- 语音播报采用老人熟悉的方言/口音，语速降低至正常80%
- 屏幕内容切换采用淡入淡出过渡，切换速度≤2秒，避免突然变化引发不适

**5.2 家属端APP操作流程（初始配置）**
```
1. 下载APP → 注册账号 → 添加设备（扫码配对）
2. 上传家庭照片（≥50张，支持从手机相册批量选择）
3. 标注家庭成员（点击人脸，输入称呼："爸爸/妈妈/小红"）
4. 录制第一条留言（语音或视频，<3分钟）
5. 设置老人常用时间表（几点睡觉、几点起床，影响内容推送策略）
→ 完成！设备即刻开始运行
```

**5.3 情绪摘要推送格式（家属APP）**
```
今日摘要 · 妈妈 · 2026年2月21日

上午 😊 状态良好，看了15张老照片
下午 😰 有些焦虑（约2小时），播放了您录制的安慰语音后改善
傍晚 😌 平静，正在听《梁祝》

💌 您有2条新留言待录制 → [立即录制]
```

### 6. 商业化路径与技术风险

**商业化路径**
- **定价**：3,980元（含首年云存储）+ 续费299元/年（无限照片+云端服务）
- **渠道**：电商（子女礼品季：父亲节/母亲节/重阳节主推）+ 社区养老服务站
- **B端**：记忆照护中心、养老院批量采购（含护理员管理平台）
- **医疗合作**：与阿尔茨海默症协会联合推广，获取专业背书

**技术风险评估**

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 老年人面部情绪识别误差大 | 高 | 大量老年人数据微调，多帧平滑，置信度过低时不触发内容变更 |
| 老人对摄像头有排斥感 | 高 | 设计为隐藏式，提供物理镜头盖，强调隐私保护 |
| 内容匹配不当引发负面情绪 | 中 | 严格的内容分类审核，家属可标注"敏感内容"（如已故亲人照片） |
| 长时间运行设备稳定性 | 中 | Android系统定时重启（凌晨3点自动重启），看门狗机制 |
| 方言TTS质量不佳 | 低 | 优先支持家属亲自录音，TTS作为补充 |
