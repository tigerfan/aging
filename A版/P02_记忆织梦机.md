# P02 记忆织梦机——睡眠期记忆巩固耳机

---

## A. 需求分析文档

### 1. 产品概述与定位

**记忆织梦机**是一款利用"靶向记忆再激活（Targeted Memory Reactivation, TMR）"技术的智能睡眠耳机，在用户慢波睡眠（SWS）阶段精准播放记忆线索音效，促进白天学习内容的记忆巩固，帮助老年人改善记忆力衰退，同时监测整夜睡眠质量。

**核心技术蓝海**：TMR是近十年神经科学重大发现（发表于Science/Nature子刊），尚无消费级产品落地，市场空白期显著。与普通助眠耳机差异化明确。

### 2. 目标用户画像

| 维度 | 描述 |
|------|------|
| 核心用户 | 60–75岁，有日常记忆困扰（忘事频繁、学新技能慢）的老年人 |
| 次要用户 | 50–60岁高知人群，主动进行认知健康管理 |
| 使用场景 | 每晚睡前佩戴，自动在睡眠中工作，无需主动操作 |
| 操作门槛 | 极低，用户只需白天"打标记"，夜间系统全自动运行 |
| 购买动机 | 改善记忆、提升睡眠质量、科技养老礼品 |

### 3. 市场痛点与机会

- 老年人记忆力下降是最普遍的老龄化困扰，现有改善手段（营养补充、脑训练游戏）效果有限且需主动坚持
- TMR技术循证基础扎实，但无产品化落地，存在明显蓝海机会
- 睡眠耳机市场（Bose Sleepbuds、小米）聚焦遮噪/音乐，无记忆增强功能
- 老年睡眠障碍与记忆衰退存在双向恶化关系，一个产品可同时解决两大痛点
- 用户无需改变生活习惯，依从性天然较高（睡觉本身是刚需）

### 4. 功能性需求（FR）

**FR-01 睡眠分期检测**
- 通过耳道内PPG传感器+单通道EEG（耳后乳突位置）检测睡眠分期
- 准确识别N1/N2/N3（慢波睡眠）/REM阶段，与PSG对比准确率≥85%
- 实时追踪整夜睡眠结构，每30秒更新一次分期结果

**FR-02 记忆线索管理（白天录入）**
- APP提供"记忆打标"功能：录制1–5秒关联音效（单词读音、提示音、音乐片段）
- 系统自动将音效与学习内容绑定（支持文字/图片/录音格式）
- 支持批量导入（CSV词汇表、日历事件等）

**FR-03 TMR智能播放**
- 检测到N3慢波睡眠后，在慢波波谷前300ms精准播放记忆线索音效（音量<40dB SPL）
- 每次SWS期间TMR播放窗口≤15分钟，不干扰睡眠深度
- 播放顺序基于Ebbinghaus遗忘曲线算法动态优化

**FR-04 睡眠质量监测**
- 统计入睡时长、各阶段占比、觉醒次数、睡眠效率评分（0–100）
- 检测打鼾（麦克风分析频谱特征）、呼吸暂停风险（SpO2+PPG）
- 每日推送睡眠报告，每周生成趋势图

**FR-05 记忆效果评估**
- 早晨唤醒后APP推送简短记忆测验（关联内容简单问答，≤8题）
- 统计TMR干预前后的记忆保留率曲线，量化改善效果
- 每月生成记忆改善综合报告

**FR-06 舒适睡眠设计**
- 主动噪声消除（ANC）功能，改善睡眠声环境
- 侧睡防压迫：超薄耳廓设计，厚度<8mm
- 检测到翻身摘耳机动作时，自动暂停播放并保存进度

### 5. 非功能性需求（NFR）

| 维度 | 指标 |
|------|------|
| 舒适性 | 单只重量<8g，耳道压力<0.5N，侧睡无压迫 |
| 续航 | 单只≥10小时（整夜），含盒充电可续充3次 |
| 音量精控 | TMR播放音量精度±1dB，确保不惊醒用户 |
| 分期准确率 | N3识别召回率≥80%，避免在非SWS期播放 |
| 数据隐私 | 睡眠录音不上传云端，仅保存分类统计结果 |
| 生物相容性 | 耳廓接触材料通过ISO 10993测试，无过敏风险 |
| 易清洁 | 耳塞可拆卸水洗，耳机主体IPX4防水 |

### 6. 约束条件与合规要求

- 定位为**健康管理设备**（非医疗器械），规避长注册周期
- TMR功能须在用户知情同意下开启，首次使用强制阅读说明书
- SpO2持续低氧检测（<88%超过5分钟）须推送就医提醒，不作诊断
- 符合蓝牙音频设备安全标准（IEC 62368-1）
- 最大输出声压级限制：用户侧≤85dB SPL，TMR模式强制≤40dB SPL

## B. 详细设计方案

### 1. 系统架构

```
[智能睡眠耳机（双耳）]
     ├── 耳道PPG（SpO2+心率）
     ├── 单通道EEG（耳后乳突电极）
     ├── 3麦克风阵列（ANC×2 + 环境采集×1）
     ├── 6轴IMU（翻身/摘除检测）
     └── 10mm动圈扬声器（精密音量控制）
          │ BLE 5.2 LE Audio
          ▼
[手机APP]
     ├── 记忆打标管理
     ├── 睡眠实时监测
     ├── 晨间记忆测验
     └── 趋势报告
          │ HTTPS/TLS 1.3
          ▼
[云端服务]
     ├── 遗忘曲线调度服务
     ├── 用户记忆内容库（加密存储）
     └── 月度报告分析
```

### 2. 硬件设计方案

**2.1 传感器组合**
- PPG：OSRAM SFH 7072（绿光+红外双波长，SpO2+心率）
- EEG：单通道干电极（耳后乳突位），配套低噪声前放ADS1119，采样率256Hz
- 麦克风：3个MEMS麦克风（前馈ANC×2 + 环境录音×1，用于打鼾检测）
- IMU：BMA400（超低功耗6轴，翻身/摘耳检测，功耗<14μA@低功耗模式）

**2.2 处理模块**
- 主控：Nordic nRF5340（低功耗BLE SoC，双核Cortex-M33）
- 音频DSP：Knowles AI（专用低功耗音频DSP，支持片上ANC，TFLite Micro推理）
- 本地存储：512MB NOR Flash（音效库+30天睡眠数据）

**2.3 结构与舒适设计**
- 耳廓贴合面：医用液态硅胶，仿耳廓三维曲面设计，厚度6.5mm
- 耳塞：记忆海绵，4种尺寸（XS/S/M/L）覆盖95%耳道尺寸
- 充电盒：磁吸定位，内置3次额外续航，支持15W无线充电输入
- 防水：主体IPX4，耳塞可拆卸水洗

**2.4 电源管理**
- 耳机单只电池：60mAh，睡眠监测模式功耗约5mW，续航≥10小时
- ANC模式额外增加约2mW功耗
- 充电盒：720mAh，可为耳机额外充电3次

**3.1 耳机固件**

**3.2 手机APP模块**

- 模型服务：睡眠分期模型定期A/B测试更新，OTA推送

### 4. 核心算法设计

**4.1 睡眠分期多模态算法**

输入特征（30秒纪元）：
- EEG：δ（0.5–4Hz）、θ（4–8Hz）、α（8–13Hz）功率比，波峰-波谷振幅
- PPG衍生：心率（HR）、RMSSD（HRV短时指标）、LF/HF比值
- IMU：体动计数（30秒内超阈值运动次数）

分类器：2层LSTM（32个隐藏单元，参数量<20K），在耳机DSP上实时运行
输出：5类（Wake/N1/N2/N3/REM），30秒分辨率，置信度评分

**4.2 慢波相位锁定（Phase-Locking TMR）**

```
算法流程：
1. 实时对EEG δ频段（0.5-2Hz）进行带通滤波
2. 计算Hilbert变换，获得瞬时相位φ(t)
3. 当分期为N3且φ(t)接近波谷（240°-300°范围）：
   - 预测波谷到达时间 t_trough
   - 在 t_trough - 300ms 时触发音效播放
   - 播放后等待至少8秒再判断下次触发
4. 每次N3期最多触发15分钟TMR，超时停止
```

**4.3 遗忘曲线调度（SM-2变体）**

- 每条记忆项维护：重复次数n、稳定性S、可提取性R = e^(-t/S)
- 当日TMR播放优先级：R值最低（即最接近遗忘）的项目优先
- 晨间测验后，根据答题正误更新S值（答对×1.3，答错×0.7）
- TMR强化系数：经TMR播放的项目额外增加30%稳定性增益

**4.4 打鼾检测算法**

- 输入：环境麦克风音频（8kHz采样率）
- 特征：梅尔频谱系数（MFCC），提取鼾声典型频率包络（50–300Hz）
- 分类：轻量CNN（MobileNet-v3 Tiny），识别打鼾vs背景噪声
- 打鼾持续>10秒时标记，结合SpO2判断呼吸暂停风险

### 5. 用户交互设计

**5.1 白天打标流程（<1分钟/条）**
```
学习新内容 → 打开APP"添加记忆" → 选择内容类型
→ 点击录音键（录制1-5秒关联音效）→ 添加内容说明（可选）
→ 保存，系统自动加入当晚TMR队列，显示"预计今晚强化"
```

**5.2 夜间（全自动，用户无感知）**
```
佩戴耳机 → 开启"睡眠守护"模式（单击充电盒盖传感器）
→ 系统自动检测分期 → SWS期自动播放记忆线索（<40dB，用户无感知）
→ 整夜监测 → 早晨闹钟由系统在REM末期轻柔唤醒
```

**5.3 晨间交互**
- 手机解锁后收到推送："今晚睡眠评分82分，TMR强化了14条记忆，来测一测？"
- 测验：卡片翻转形式，每题≤10秒，共5–8题，答对有动画鼓励
- 全程<3分钟，不影响早晨节奏

**5.4 适老化设计重点**
- APP字体全局≥18pt，关键数字≥28pt
- 操作步骤最多3步，无二级菜单嵌套
- 颜色：主色调暖橙，非纯蓝（老年人蓝色辨别力弱）
- 错误容忍：误触有撤销提示，不直接删除数据

### 6. 商业化路径与技术风险

**商业化路径**
- **阶段1**（0–12个月）：与睡眠医学中心合作临床验证（≥100名老年受试者），积累循证数据
- **阶段2**（12–24个月）：定价3,200元（硬件）+ 订阅69元/月（无限记忆条目+月度报告）
- **渠道**：电商DTC + 老年大学/社区课程配套推广 + 子女礼品场景
- **B端**：养老院记忆训练项目团购，含护理人员管理平台
- **阶段3**：与健康险合作，纳入认知健康管理保障计划

**技术风险评估**

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| EEG单通道睡眠分期精度不足 | 高 | 多模态融合（PPG+IMU+EEG），降低单一依赖 |
| TMR效果个体差异大 | 中 | 2周自适应校准期，个性化调整播放时机和音量 |
| 侧睡舒适度投诉 | 中 | 提供5款不同耳廓形态的贴合模具，用户自选 |
| 老年人不愿每天"打标"（依从性） | 中 | 与手机日历/健康APP联动，自动导入日程内容 |
| 商业医保准入谈判周期长 | 低 | 先自费市场，同步开展卫生经济学研究 |
