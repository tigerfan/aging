# P30 照护协同仪——多方协同护理管理系统

---

## A. 需求分析文档

### 1. 产品概述与定位

**照护协同仪**是一套连接老人、家属、居家护理员、社区医护和医院医生的多方协同照护管理平台，通过护理任务调度、实时状态共享、远程指导和照护质量评估，解决老年居家照护中信息孤岛、责任不清、质量无法评估等核心问题。硬件端为护理员配备的智能照护手环（操作引导+数据录入），软件端覆盖老人、家属、护理员、医护四类角色，构建中国老龄化社会最需要的"照护闭环"。

**蓝海点**：中国居家护理市场快速增长（超500亿/年），但服务质量管理极度落后，家属与护理员之间信任危机普遍；照护协调软件（care coordination）在欧美已成熟（Caremerge、PointClickCare），但在中国几乎空白；结合硬件的解决方案更是零。

### 2. 目标用户画像

| 维度 | 描述 |
|------|------|
| 老人 | 75–90岁，半自理或完全失能，居家接受专业护理 |
| 家属 | 在外工作的子女，雇用护理员，希望实时了解照护质量 |
| 护理员 | 居家护理员（保姆/护工），需要专业操作指导和记录工具 |
| 社区护士 | 社区医院/卫生服务中心的护士，提供定期上门护理指导 |
| 医院医生 | 老人的主治医生，需了解居家护理状态以指导治疗 |

### 3. 市场痛点与机会

- 中国失能老人超4,000万，居家照护需求巨大但专业化程度极低
- 家属最大痛点：雇用的护理员"不知道在干什么"，无法评估护理质量
- 护理员最大痛点：工作内容无法记录，责任划分不清，专业能力提升无途径
- 老人最大痛点：护理员不懂医疗操作（如压疮护理、鼻饲管理），安全风险高
- 护理员短缺是全球性问题，AI辅助护理员提升效率和专业性是关键解法

### 4. 功能性需求（FR）

**FR-01 护理任务管理系统**
- 任务模板库：覆盖居家护理全场景（协助进食/翻身/褥疮护理/用药/康复训练/洗澡等，50+模板）
- 个性化护理计划：医生/护士根据老人情况生成个性化护理计划（任务类型/频次/注意事项）
- 任务调度：每日护理任务清单，优先级排序，时间提醒
- 任务完成确认：护理员完成每项任务后打卡（含照片/视频记录，防止虚报）

**FR-02 护理员智能手环**
- 操作引导：手环振动提醒下一项护理任务，配合语音/屏幕引导操作步骤
- 一键打卡：完成任务按按键确认，自动记录时间戳
- 应急求助：检测到老人状况异常（心率/跌倒，通过P24/P22等联动），手环振动告警
- 护理数据录入：声控录入简短观察（"老人今日进食80%，精神状态一般"），AI自动转录

**FR-03 实时状态共享**
- 家属视角：实时查看护理员当前位置（在岗/外出）、今日护理任务完成情况
- 老人状态日报：自动汇总今日健康指标（P13–P27各设备数据）+ 护理员观察记录
- 视频巡视（可选）：家属可申请连接居室监控摄像头（须老人同意授权）
- 护理日志：可检索历史护理记录（照片、操作记录、观察备注）

**FR-04 专业护理指导**
- 操作视频库：200+居家护理操作标准视频（翻身、褥疮护理、鼻饲、留置导尿管护理等）
- 在线培训：护理员在职培训课程，完成后获得护理员技能认证
- 社区护士远程指导：复杂操作前，护理员可发起视频请求社区护士远程指导
- AI操作识别：手环摄像头录制操作过程，AI评估操作规范性（实验性功能）

**FR-05 照护质量评估**
- 自动评分：任务按时完成率、异常事件响应速度、观察记录完整度
- 健康结果导向：老人健康指标趋势（与照护质量关联分析）
- 家属评价：每周家属对护理员服务评分（1–5星）
- 护理员个人档案：积累评分、认证、从业经历，形成可信护理员档案（类似滴滴司机档案）

**FR-06 紧急事件处理**
- 分级告警：轻微异常（通知家属）→ 较严重（通知护理员+家属）→ 紧急（自动拨打120）
- 事件记录：告警时间、处置措施、结果，形成事件闭环记录
- 复盘分析：每次紧急事件后AI辅助分析原因和预防措施

### 5. 非功能性需求（NFR）

| 维度 | 指标 |
|------|------|
| 任务提醒及时性 | 护理时间到达前15分钟手环振动提醒 |
| 手环续航 | ≥3天（护理员佩戴，每日8小时工作） |
| 操作简单性 | 护理员（40–55岁，非高学历）独立上手时间<30分钟 |
| 离线可用性 | 网络断开时，手环继续记录任务完成情况，联网后同步 |
| 视频通话质量 | 720p/30fps（兼容4G网络） |
| 隐私管理 | 护理记录（含照片/视频）家属可查，护理员无权删除 |

### 6. 约束条件与合规要求

- 护理员位置追踪：需护理员知情同意（劳动法合规），仅记录在岗时段
- 老人监控摄像：须经过老人本人明确授权，随时可撤销
- 护理操作AI评估：仅供内部质量提升用，不用于劳动纠纷证据（须法律咨询）
- 护理员个人数据：按《个人信息保护法》保护，数据所有权属于护理员本人
- 医疗操作指导：远程指导复杂医疗操作须有执照护士参与，不由AI单独指导

### 2. 硬件设计方案

**2.1 护理员手环**
- 主控：Nordic nRF5340（低功耗，BLE 5.0）
- 显示：1.5英寸圆形AMOLED（360×360），显示任务名称和时间
- 振动：LRA马达（强提醒模式）
- 麦克风：MEMS单麦克风（语音备注录入，>0.5m有效拾音）
- NFC：M24LR系列（工牌感应，老人家门NFC贴片打卡——"到岗/离岗"确认）
- 电池：250mAh，3天续航（8小时/天）
- 结构：运动腕带风格，IP67防水（护理操作频繁接触液体）

**2.2 老人家门NFC贴片**
- 被动NFC标签（无需电源），贴于门框
- 护理员刷手环 → 记录"到岗/离岗"时间戳（防止虚报工时）
- 成本：每片<3元，随系统免费提供

### 3. 软件设计方案

**3.1 护理任务引擎**
```python
class CareTaskEngine:
    def generate_daily_tasks(self, resident_id: str, date: date) -> List[CareTask]:
        # 从护理计划中获取当日任务
        care_plan = self.db.get_care_plan(resident_id)
        tasks = []
        
        for template in care_plan.task_templates:
            if template.is_due(date):
                task = CareTask(
                    template=template,
                    scheduled_time=template.compute_scheduled_time(date),
                    priority=template.priority,
                    notes=template.personal_notes  # 个性化注意事项
                )
                tasks.append(task)
        
        return sorted(tasks, key=lambda t: t.scheduled_time)
    
    def complete_task(self, task_id: str, completion_data: dict):
        task = self.db.get_task(task_id)
        task.completed_at = datetime.now()
        task.photos = completion_data.get('photos', [])
        task.observations = completion_data.get('observations', '')
        task.quality_score = self.assess_quality(task, completion_data)
        
        # 通知家属
        self.notify_family(task, completion_data)
        self.db.save(task)
```

**3.3 照护质量评分算法**
```python
def compute_care_quality_score(resident_id, period='weekly'):
    tasks = get_completed_tasks(resident_id, period)
    
    # 维度1：任务按时完成率（30%）
    on_time_rate = sum(1 for t in tasks if t.is_on_time) / len(tasks)
    
    # 维度2：记录完整度（照片/备注覆盖率）（25%）
    record_completeness = sum(1 for t in tasks if t.has_record) / len(tasks)
    
    # 维度3：异常事件响应速度（25%）
    avg_response_time = mean([e.response_time_min for e in get_alerts(period)])
    response_score = max(0, 1 - avg_response_time/60)  # 60分钟响应=0分
    
    # 维度4：家属满意度（20%）
    family_rating = get_family_rating(resident_id, period) / 5.0
    
    total_score = (on_time_rate*0.30 + record_completeness*0.25 + 
                   response_score*0.25 + family_rating*0.20) * 100
    return round(total_score, 1)
```

### 4. 用户交互设计

**护理员手环日常流程**
```
早晨刷手环到老人家门NFC → 到岗打卡
→ 手环振动+显示"第1项任务：协助早餐（07:30）"
→ 完成早餐后按手环按键 → 手环显示"请简单描述老人今早进食情况"
→ 护理员口述（"大约吃了7成，精神还好"）→ 语音录入成功
→ 显示下一任务...
→ 离岗时刷NFC → 离岗打卡 → 手环显示"今日任务完成率100%，棒！"
```

**家属APP日常体验**
```
早起推送："爸爸今天已完成早餐护理，护理员记录进食良好"
每晚推送：今日护理摘要（健康数据+护理完成情况+护理员备注）
周度推送：护理员本周质量评分（任务完成率/家属评分/异常响应）
出现异常：立即推送（"护理员发现老人今日皮肤有异常，已联系社区护士"）
```

**社区护士端（Web）**
- 待处理请求列表（护理员发起的远程指导请求）
- 老人健康档案概览（P29平台整合数据）
- 月度质量评估报告（辖区内各老人护理质量综合评分）

### 5. 商业化路径与技术风险

**商业化路径**
- **SaaS平台**：家属版239元/年（管理1位老人，含手环）
- **护理员认证**：护理员培训认证课程，299元/人（与护理员平台资质认证挂钩）
- **机构版**：养老院/护理站管理平台，3,000元/月（管理100人以下），大型机构定制
- **护理员市场**：平台积累认证护理员档案 → 家属在平台雇用护理员（撮合佣金15%）
- **政府购买**：长期护理险（全国试点，政策支撑）配套评估和记录工具

**技术风险评估**

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 护理员（40–55岁）不愿学APP | 高 | 核心功能靠手环完成（实体按键，无需看屏幕），手机APP作为辅助 |
| 任务打卡弄虚作假（补打卡） | 高 | NFC到岗确认+任务时间窗口限制（提前/延后超过2小时无效） |
| 家属监控导致护理员抵触 | 中 | 监控内容家属可见，但护理员也可查看（透明双向），建立信任而非监控 |
| 照护质量评分被钻空子（完成任务但质量差） | 中 | 结合老人健康结果（体重、褥疮面积变化等客观指标）综合评估 |
| 护理照片/视频存储成本 | 中 | 照片压缩存储（WebP，90%压缩率），视频仅保留30秒缩略片段 |
