# 产品21：老年驾驶认知辅助座舱插件（详细设计实现文档）

## 0. 文档信息
- 版本：v1.0
- 目标对象：仍有短途驾驶需求的老年驾驶者
- 目标：降低复杂路况下的认知负荷与决策失误

## 1. 建设目标
1. 在存量车辆中低成本加装认知辅助能力。
2. 提供“短句、单指令”的简化提示。
3. 形成驾驶风险趋势评估和家属守护机制。

## 2. 范围边界
- **In Scope**：风险识别、简化导航、提示与日志。
- **Out of Scope**：自动驾驶接管、保险定责系统替代。

## 3. 功能需求
### Must
- FR-01：高负荷场景识别（复杂路口、并线、夜间）。
- FR-02：简化提示输出（语音/抬头显示/方向盘振动）。
- FR-03：危险场景建议“减速/靠边”。

### Should
- FR-04：家属守护模式与异常提醒。
- FR-05：驾驶能力趋势分析。

### Could
- FR-06：保险联动服务。

## 4. 非功能指标
- 风险提示时延 < 300ms。
- 误提示率 < 8%。
- 车规电磁兼容与安装安全达标。

## 5. 总体架构
```text
前装摄像头 + OBD接口 + IMU
   -> 边缘计算盒(感知+负荷估计)
      -> 抬头提示器/方向盘振动模块
         -> 家属守护App与日志服务
```

## 6. 硬件设计
- 广角摄像头：前向路况感知。
- OBD采集：车速、转向、制动等信号。
- 抬头提示器：减少视线偏移。
- 振动模块：在噪声环境下增强提醒触达。

## 7. 软件模块
1. `ScenePerception`：目标检测与车道事件识别。
2. `CognitiveLoadEstimator`：认知负荷建模。
3. `GuidanceSimplifier`：提示语压缩。
4. `AlertOutputManager`：多通道提醒编排。
5. `DriveTrendService`：驾驶趋势报告。

## 8. 数据模型
- `RiskScenarioEvent { type, severity, ts }`
- `CognitiveLoadScore { value, factors[] }`
- `GuidanceAction { text, channel, priority }`
- `DriveTrendReport { week, near_miss_count, trend }`

## 9. 算法策略
- 认知负荷估计融合：路况复杂度 + 操作频率 + 时间环境。
- 提示策略：高负荷时仅保留关键单指令。
- 抑制策略：短时重复场景去重避免提示疲劳。

## 10. 接口定义
- `GET /api/v1/drive/load/latest`
- `POST /api/v1/drive/alert/ack`
- `GET /api/v1/drive/trend/report`

## 11. 交互流程
1. 系统持续感知路况与驾驶行为。
2. 负荷升高时触发简化提示。
3. 用户确认后继续行驶，异常持续则建议靠边。
4. 行程结束生成摘要报告。

## 12. 安全与合规
- 系统仅辅助提示，不接管车辆控制。
- 视频本地滚动缓存，默认不上传。
- 车规安装要求与责任边界明示。

## 13. 测试与验收
- 封闭道路场景测试（夜间/雨天/并线）。
- 提示可理解性测试（老年用户）。
- 验收：近失误事件率下降 > 20%。

## 14. 实施计划
- P1：封闭道路验证。
- P2：城市路测试点。
- P3：车后市场规模化安装。

## 15. 风险与缓解
- 风险：误提示干扰驾驶。  
  缓解：多传感一致性门限 + 抑制策略。
