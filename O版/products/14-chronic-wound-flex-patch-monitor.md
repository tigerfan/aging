# 产品14：慢性伤口与压疮柔性监测贴（详细设计实现文档）

## 0. 文档信息
- 版本：v1.0
- 适用对象：长期卧床、糖尿病足高风险老人
- 目标：让“数字敷料”具备连续监测与护理决策支持能力

## 1. 建设目标
1. 持续监测伤口温湿与渗液趋势。
2. 提前识别感染恶化风险。
3. 输出可执行换药时机建议。

## 2. 范围边界
- **In Scope**：贴片监测、风险评分、随访任务。
- **Out of Scope**：替代临床清创、治疗方案制定。

## 3. 功能需求
### Must
- FR-01：渗液量趋势监测。
- FR-02：感染风险提醒。
- FR-03：换药时机建议。

### Should
- FR-04：护理员远程随访。
- FR-05：伤口档案持续记录。

### Could
- FR-06：图谱归档与长期疗效分析。

## 4. 非功能指标
- 贴片连续佩戴完整率 > 85%。
- 感染预警灵敏度 > 90%。
- 皮肤刺激不良事件率 < 1%。

## 5. 架构设计
```text
柔性贴片(温湿/渗液/电学)
   -> 近场读取器(手机或护理终端)
      -> 护理云平台(风险评估+工单)
```

## 6. 硬件设计
- 柔性电极：贴合皮肤曲面。
- 温湿度微传感：局部环境持续采样。
- 低功耗蓝牙标签：短距数据传输。
- 医用胶体：保证生物相容性。

## 7. 软件模块
1. `PatchDataCollector`：采样与质量校验。
2. `InflammationScorer`：炎症风险指数计算。
3. `DressingAdvisor`：换药建议生成。
4. `NursingTaskEngine`：随访与工单管理。

## 8. 数据模型
- `WoundReading { ts, temp, moisture, exudate_index }`
- `InflammationIndex { score, level, reasons[] }`
- `DressingSuggestion { urgency, recommended_window }`
- `FollowupTask { assignee, due_time, status }`

## 9. 算法策略
- 多指标融合：温度梯度 + 湿度趋势 + 渗液变化。
- 分层建议：观察、换药、复诊三级。
- 批次偏差校正：按贴片批次做漂移补偿。

## 10. 接口定义
- `POST /api/v1/wound/reading/upload`
- `GET /api/v1/wound/risk/latest`
- `POST /api/v1/wound/task/create`
- `GET /api/v1/wound/timeline`

## 11. 交互流程
1. 贴片佩戴后自动采样。
2. 护理端查看风险等级与建议。
3. 达阈值自动创建换药或复诊任务。
4. 护理员回填处置结果形成闭环。

## 12. 安全与合规
- 医学数据加密与访问留痕。
- 贴片批次可追溯，支持召回机制。
- 明确监测辅助属性，不替代医嘱。

## 13. 测试与验收
- 生物相容性测试。
- 渗液模拟液对照测试。
- 验收：感染预警灵敏度 > 90%，任务闭环率 > 85%。

## 14. 实施计划
- P1：院内康复科验证。
- P2：家庭护理随访试点。
- P3：机构规模化部署。

## 15. 风险与缓解
- 风险：贴片脱落导致监测中断。  
  缓解：佩戴完整性检测 + 提醒更换。
- 风险：护理执行不一致。  
  缓解：标准化任务卡与过程审计。
