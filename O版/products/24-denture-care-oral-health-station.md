# 产品24：义齿护理与口腔健康站（详细设计实现文档）

## 0. 文档信息
- 版本：v1.0
- 目标用户：佩戴活动义齿老人
- 目标：把“清洁动作”升级为“口腔健康管理流程”

## 1. 建设目标
1. 自动化义齿清洁与消毒，减少操作不规范。
2. 追踪磨损与炎症风险趋势。
3. 建立复诊提醒与长期档案。

## 2. 范围边界
- **In Scope**：清洁消毒、形变评估、风险提醒、复诊管理。
- **Out of Scope**：口腔疾病诊断、治疗替代。

## 3. 功能需求
### Must
- FR-01：一键清洁消毒流程。
- FR-02：佩戴时长与习惯记录。
- FR-03：磨损与炎症风险提醒。

### Should
- FR-04：复诊提醒与历史比对。
- FR-05：医生咨询入口。

### Could
- FR-06：耗材订阅自动补货。

## 4. 非功能指标
- 清洁流程成功率 >= 98%。
- 杀菌模块寿命可追踪。
- 日常操作步骤 <= 2步。

## 5. 总体架构
```text
护理站(超声+UV+形变扫描)
   -> 风险分析引擎
      -> 口腔健康App/医生协作端
```

## 6. 硬件设计
- 超声清洁模块：去除附着残留。
- UV消毒模块：联锁防误照射。
- 形变扫描组件：义齿磨损趋势分析。
- 语音提示：帮助低视力用户操作。

## 7. 软件模块
1. `CareCycleController`：清洁流程控制。
2. `WearAnalysisEngine`：形变对比与趋势估计。
3. `InflammationRiskScorer`：炎症风险计算。
4. `ClinicReminderService`：复诊计划管理。

## 8. 数据模型
- `DentureCareCycle { start_ts, end_ts, status }`
- `WearDelta { region, delta_value, trend }`
- `InflammationRisk { level, reasons[] }`
- `ClinicReminder { due_date, clinic_info, status }`

## 9. 算法策略
- 形变比对：当前扫描与历史模板对齐。
- 风险融合：佩戴时长异常 + 清洁不足 + 形变偏移。
- 提醒策略：轻提醒、复查提醒、尽快就诊三级。

## 10. 接口定义
- `POST /api/v1/denture/care/start`
- `GET /api/v1/denture/risk/latest`
- `GET /api/v1/denture/wear/trend`
- `POST /api/v1/denture/clinic/reminder`

## 11. 交互流程
1. 睡前放入义齿，系统自动完成清洁消毒。
2. 次晨取出前给出佩戴建议。
3. 异常趋势触发复诊提醒。

## 12. 安全与合规
- 材料与消毒流程符合相关标准。
- 健康记录默认本地保存，可授权导出。
- 设备提示避免夸大医疗效果。

## 13. 测试与验收
- 清洁效率与杀菌效果测试。
- 老人操作可用性测试。
- 验收：清洁执行率>85%，炎症复发率下降。

## 14. 实施计划
- P1：护理机构场景试点。
- P2：家庭消费版上线。
- P3：医生协作服务扩展。

## 15. 风险与缓解
- 风险：操作复杂导致闲置。  
  缓解：一键流程 + 语音指导。
