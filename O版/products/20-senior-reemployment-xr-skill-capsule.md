# 产品20：适老再就业XR技能镜像舱（详细设计实现文档）

## 0. 文档信息
- 版本：v1.0
- 目标：帮助55+人群恢复岗位技能与就业信心
- 场景：社区培训中心、职业服务机构

## 1. 建设目标
1. 复刻真实岗位流程而非泛娱乐VR训练。
2. 提供错误纠正与胜任力评估。
3. 打通培训结果与企业岗位需求匹配。

## 2. 范围边界
- **In Scope**：岗位模拟、能力评分、岗位推荐。
- **Out of Scope**：实际招聘决定、法律用工替代。

## 3. 功能需求
### Must
- FR-01：岗位流程模拟训练。
- FR-02：错误识别与实时纠正。
- FR-03：能力雷达图输出。

### Should
- FR-04：企业岗位画像匹配。
- FR-05：课程进度管理。

### Could
- FR-06：远程面试联动。

## 4. 非功能指标
- 训练完成率 > 80%。
- 晕动不适率 < 5%。
- 评分稳定性（重测一致性）> 0.85。

## 5. 总体架构
```text
XR终端(可选) + 实体操作台
   -> 岗位模板引擎
      -> 能力评估系统
         -> 就业平台匹配服务
```

## 6. 硬件设计
- XR头显（可选）：支持坐姿训练。
- 触觉反馈模块：模拟岗位操作触感。
- 模块化外设：扫码枪、称重台、分拣面板等。

## 7. 软件模块
1. `JobTemplateEngine`：岗位流程建模。
2. `ActionEvaluator`：动作质量评分。
3. `ErrorExplainer`：错因解释与重练建议。
4. `CompetencyProfiler`：胜任力向量生成。
5. `JobMatcher`：岗位推荐与匹配。

## 8. 数据模型
- `JobTemplate { role, steps[], quality_rules }`
- `SkillSession { user_id, template_id, metrics }`
- `ErrorTaxonomy { type, severity, fix_plan }`
- `CompetencyVector { speed, accuracy, safety, consistency }`

## 9. 算法策略
- 任务分解：步骤级时间与质量双指标。
- 评分机制：动作完成度 + 安全合规权重。
- 匹配机制：能力向量与岗位画像余弦匹配。

## 10. 接口定义
- `POST /api/v1/job/template/publish`
- `POST /api/v1/skill/session/start`
- `GET /api/v1/skill/report/{session_id}`
- `GET /api/v1/job/recommend?user_id=`

## 11. 交互流程
1. 选择目标岗位课程。
2. 进入XR/实体混合训练。
3. 实时提示错误与纠正方法。
4. 训练结束生成能力雷达和岗位建议。

## 12. 安全与合规
- 防眩晕策略：低运动视差、可坐姿。
- 就业数据授权可撤回。
- 评分结果仅作推荐，不作强制筛选依据。

## 13. 测试与验收
- 岗位模板真实度评审（企业方参与）。
- 用户晕动与易用性测试。
- 验收：课程完成率>80%，匹配准确率提升。

## 14. 实施计划
- P1：零售/客服岗位。
- P2：仓配/轻装配岗位。
- P3：企业共建标准化模板库。

## 15. 风险与缓解
- 风险：企业认可度不足。  
  缓解：联合龙头企业共建评价标准。
- 风险：老年用户恐惧XR。  
  缓解：先实体台训练，再渐进进入XR。
