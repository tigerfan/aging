# 产品18：孤独风险预测与兴趣共创桌（详细设计实现文档）

## 0. 文档信息
- 版本：v1.0
- 目标：把“被动聊天”转为“任务驱动社交共创”
- 适用场景：社区活动室、居家客厅

## 1. 建设目标
1. 识别孤独风险变化并主动推荐活动。
2. 通过共创任务提升持续社交参与度。
3. 打通社区运营与线上协作闭环。

## 2. 范围边界
- **In Scope**：任务匹配、共创互动、运营管理、反骚扰。
- **Out of Scope**：医疗心理诊断、强制社交干预。

## 3. 功能需求
### Must
- FR-01：支持拼图/故事/手工等共创任务联机。
- FR-02：活动匹配与队友推荐。
- FR-03：积分激励与进度追踪。

### Should
- FR-04：情绪趋势识别与活动个性化推荐。
- FR-05：主持人工具与活动复盘。

### Could
- FR-06：跨代参与（孙辈协作）。

## 4. 非功能指标
- 月活留存 > 55%。
- 活动匹配成功率 > 80%。
- 反骚扰处置时延 < 30min。

## 5. 总体架构
```text
交互桌终端(触控+触觉)
   -> 内容与匹配平台
      -> 社区运营后台
         -> 家庭/跨代协作App
```

## 6. 硬件设计
- 大尺寸触控屏：低门槛交互。
- 触觉反馈模块：增强参与感。
- 实体扩展接口：手工模块连接。
- 阵列麦克风：语音辅助操作。

## 7. 软件模块
1. `ActivityOrchestrator`：活动编排。
2. `MatchingEngine`：同好匹配。
3. `EngagementAnalyzer`：参与度评估。
4. `RewardSystem`：积分与成就体系。
5. `SafetyModeration`：反骚扰与内容审核。

## 8. 数据模型
- `CoCreateSession { task_type, members, duration, outcome }`
- `EngagementScore { user_id, weekly_score, trend }`
- `RecommendationPlan { candidate_tasks[], reasons[] }`
- `SafetyReport { reporter, type, severity }`

## 9. 算法策略
- 联合建模：参与频次 + 会话质量 + 情绪信号。
- 推荐目标：优先低门槛高反馈任务。
- 冷启动：按兴趣问卷 + 社区画像初始化。

## 10. 接口定义
- `POST /api/v1/activity/create`
- `GET /api/v1/activity/recommend`
- `POST /api/v1/activity/join`
- `POST /api/v1/safety/report`

## 11. 交互流程
1. 每日推送可参与共创任务。
2. 用户一键加入或发起任务。
3. 任务中实时协作与语音引导。
4. 完成后获得积分与社交反馈。

## 12. 安全与合规
- 提供退出机制与黑名单。
- 不做过度情绪操控。
- 社交身份分层可见（昵称/实名分离）。

## 13. 测试与验收
- 可用性测试（老年交互易学性）。
- 匹配效果A/B测试。
- 验收：孤独感量表均值改善，月活留存>55%。

## 14. 实施计划
- P1：社区活动室试点。
- P2：家庭延伸版。
- P3：跨代协作生态。

## 15. 风险与缓解
- 风险：新鲜感下降。  
  缓解：季节主题 + 真人主持 + 活动轮换。
- 风险：社交冲突。  
  缓解： moderation规则 + 快速干预流程。
